Defense Capabilities

Six layers of protection for your credentials

Fernet encryption

AES-128-CBC via Fernet (includes HMAC). Keys from PBKDF2-HMAC-SHA256 with 600,000 iterations — OWASP 2023 aligned.

Zero Knowledge

Your master password is never stored. Only a salted hash exists. Not even the tool can read it.

Hardware Bound

Encryption keys derived from your machine's hardware identifiers. Data is tied to your device.

Brute Force Shield

Adaptive lockout system with progressive security. Each failed attempt tightens the defense.

Session Protection

Automatic timeout after 30 minutes of inactivity. Sensitive operations require re-authentication.

Local Only

No network. No cloud. No telemetry. Everything stays on your machine. Period.

Security levels

Choose once at first-time setup (irondome create bunker).
ModeHow it worksBest for
Biometric onlyTouch ID / Windows Hello / fingerprint unlocks vault key from the OS keychain.Speed — single factor.
Biometric + passwordBiometric gate plus master password derives keys via PBKDF2.Maximum security.
Password onlyMaster password derives all keys via PBKDF2.Universal compatibility.

Terminal UI

Primary interface: a Textual-based TUI. Launch with irondome (default).

  • 12 screens — splash, login, dashboard, vault browser, entry detail, generator, save form, settings, backup, status, help, confirmations.
  • Keyboard-driven — arrows, Enter, Esc, Tab, hotkeys; Ctrl+P command palette.
  • Security-first — masked input, alternate screen (limits scrollback exposure), clipboard auto-clear, signal handlers, memory protection.

Headless and scripting: use the irondome / bunker commands in Command Center above (same names as the README).

Vault directory layout

~/.password_manager/
├── password_manager.log           # Non-sensitive audit trail
├── settings.json                  # User preferences
├── backups/
│   └── .passwords_backup_*.enc    # Encrypted backups
└── secrets/                       # chmod 0700
    ├── .passwords.enc             # Encrypted vault (AES-128-CBC)
    ├── salt.bin                   # Key derivation salt
    ├── .master_user.enc           # Encrypted master user
    ├── .master_hash.enc           # Encrypted PBKDF2 hash
    ├── .login_attempts.dat        # Per-device lockout counter
    └── .airspace.session          # Active session (0600)
